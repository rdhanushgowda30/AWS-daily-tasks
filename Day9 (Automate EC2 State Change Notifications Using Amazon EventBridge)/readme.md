# Task 9: Automate EC2 State Change Notifications Using Amazon EventBridge

## Project Overview

This project demonstrates how to automate notifications for Amazon EC2 instance state changes using Amazon EventBridge and Amazon SNS. Whenever an EC2 instance changes its state (Start, Stop, Reboot, or Terminate), EventBridge captures the event and forwards it to an Amazon SNS topic, which sends an email notification to subscribed users. This automation enables real-time monitoring of infrastructure events without manual intervention.

---

# Objective

- Create an Amazon EventBridge rule.
- Monitor EC2 instance state change events.
- Configure Amazon SNS as the notification target.
- Receive email notifications whenever an EC2 instance changes state.
- Verify the automation by stopping and starting an EC2 instance.

# Step 2: Create an EventBridge Rule

Navigate to

```
EventBridge
    → Rules
```

Click

```
Create Rule
```

Configure the rule.

| Setting | Value |
|----------|-------|
| Rule Name | EC2-State-Notification |
| Description | Send an email whenever an EC2 instance changes state |
| Event Bus | Default |
| Rule Type | Rule with an Event Pattern |

Click

```
Next
```

### Why Use the Default Event Bus?

The **Default Event Bus** automatically receives events from AWS services.

Whenever an EC2 instance changes state, AWS publishes an event to this event bus.

# Step 7: Understand the Event Flow

When the EC2 instance is stopped, AWS automatically generates an event.

Example Event

```json
{
  "source": "aws.ec2",
  "detail-type": "EC2 Instance State-change Notification",
  "detail": {
    "state": "stopped"
  }
}
```

This event is generated automatically by AWS.

The complete workflow is:

```
EC2 Instance
        │
        ▼
AWS Generates Event
        │
        ▼
Default Event Bus
        │
        ▼
EventBridge Rule
        │
        ▼
Amazon SNS Topic
        │
        ▼
Email Notification
```

---

# Step 8: Verify Email Notification

Within approximately one minute, an email notification should arrive.

The notification includes information such as:

- EC2 Instance ID
- Previous State
- Current State
- Event Time

Example

```
EC2 Instance State Change

Instance ID:
i-0abcd1234ef567890

Current State:
stopped
```

### Expected Result

An email notification confirming the EC2 instance state change is received.
